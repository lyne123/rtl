# 🔧 延迟链设计修正说明

## 📋 修正背景

在设计审查过程中发现原60级CARRY4延迟链设计存在量程不足的问题：
- **原设计**: 60级 × 35ps = 2.1ns < 2.5ns (粗计数周期) ❌
- **要求**: 细计数量程必须 ≥ 2.5ns ✓

## 🎯 修正内容

### 1. 延迟链参数修正

| 参数 | 原设计 | 修正后 | 说明 |
|------|--------|--------|------|
| CARRY4级数 | 60级 | 72级 | 满足2.5ns量程要求 |
| 总延迟 | 2.1ns | 2.52ns | >2.5ns ✓ |
| 理论精度 | 41.7ps | 34.7ps | 2.5ns/72级 |
| 温度计码宽度 | 60位 | 72位 | 对应延迟级数 |
| 解码器输出 | 6位 | 7位 | 2^7=128>72 |

### 2. 修改文件清单

#### ✅ 已完成修改
- `tdc_top_carry4.v` - 顶层模块参数更新
- `carry4_delay_chain.v` - 延迟链级数更新
- `thermometer_decoder.v` - 输入/输出位宽更新
- `timestamp_synthesizer.v` - 细计数参数更新

### 3. 精度重新计算

#### 理论精度
```
理论分辨率 = 粗计数周期 / 延迟级数
           = 2.5ns / 72
           = 34.7ps
```

#### 实际精度 (考虑误差)
```
误差来源:
- CARRY4延迟变化: ±10% → ±3.5ps
- 时钟抖动: ±20ps
- 温度漂移: ±5%
- 工艺偏差: ±15%

总精度 ≈ 50-100ps
```
