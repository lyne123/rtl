# TDC v5 直接采样架构设计文档

## 1. 设计概述

### 1.1 版本信息
- **版本号**: v5
- **架构名称**: 直接采样架构 (Direct Sampling Architecture)
- **设计目标**: 0-50ns脉宽测量，35ps分辨率
- **适用场景**: 超窄脉冲测量、高精度时间间隔测量

### 1.2 核心创新
传统TDC依赖边沿检测模块，无法测量小于10ns的脉宽。v5版本采用**直接延迟链采样**技术，突破这一限制，实现：
- **全范围测量**: 0-50ns无缝覆盖
- **超高精度**: 35ps理论分辨率
- **无盲区检测**: 可测量任意窄的脉宽

## 2. 系统架构

### 2.1 整体架构
```
PWM输入 → 延迟链采样 → 温度计码解码 → 数据融合 → 时间间隔输出
            ↓
        粗计数器 → 数据融合
            ↓
        系统时钟(400MHz)
```

### 2.2 模块组成

#### 2.2.1 PWM延迟链采样模块 (pwm_delay_chain_sampler.v)
- **功能**: 直接将PWM信号送入72级CARRY4延迟链
- **采样方式**: 400MHz时钟边沿采样延迟链状态
- **输出**: 72位温度计码
- **特点**: 三级同步，减少亚稳态风险

#### 2.2.2 粗计数器模块 (coarse_counter_pwm.v)
- **功能**: 测量PWM脉宽的粗计数部分
- **时钟**: 400MHz (2.5ns分辨率)
- **范围**: 1-20个时钟周期 (2.5ns-50ns)
- **特点**: 边沿检测，错误处理

#### 2.2.3 温度计码解码器 (thermometer_decoder_72to7.v)
- **功能**: 72位温度计码 → 7位二进制
- **算法**: 分组加法树
- **精度**: 1级延迟 (35ps)
- **特点**: 组合逻辑，低延迟

#### 2.2.4 数据融合模块 (pwm_data_fusion.v)
- **功能**: 融合延迟链和粗计数结果
- **策略**: 智能选择测量模式
- **精度**: 保持35ps分辨率
- **特点**: 错误检测，边界处理

#### 2.2.5 顶层模块 (tdc_top_v5.v)
- **功能**: 系统集成和接口管理
- **时钟**: MMCM生成400MHz时钟
- **接口**: 标准TDC接口
- **特点**: 模块化设计，易于扩展

## 3. 测量原理

### 3.1 短脉宽测量 (0-2.5ns)
```
测量值 = 温度计码中1的个数 × 35ps
```
- **原理**: PWM信号在延迟链中传播，1的个数与传播距离成正比
- **精度**: 35ps (单级CARRY4延迟)
- **示例**: 25个1 → 25 × 35ps = 875ps

### 3.2 长脉宽测量 (2.5ns-50ns)
```
测量值 = 粗计数值 × 2500ps + 细计数值 × 35ps
```
- **原理**: 粗计数提供大范围，细计数提供高精度
- **精度**: 保持35ps分辨率
- **示例**: 粗计数=5, 细计数=25 → 5×2500ps + 25×35ps = 13.375ns

### 3.3 数据融合策略

| 脉宽范围 | 测量模式 | 精度 | 特点 |
|----------|----------|------|------|
| 0-2.5ns | 纯延迟链 | 35ps | 避免量化误差 |
| 2.5ns-50ns | 粗+细组合 | 35ps | 扩展测量范围 |
| 错误情况 | 降级模式 | 可变 | 保证系统稳定 |

## 5. 设计优势

### 5.1 与传统TDC对比

| 特性 | 传统TDC | v5直接采样TDC |
|------|----------|---------------|
| **最小脉宽** | >10ns | 无限制 |
| **测量原理** | 边沿检测 | 直接采样 |
| **精度** | 35ps | 35ps |
| **复杂度** | 中等 | 较高 |
| **适用场景** | 通用 | 超窄脉冲 |

### 5.2 技术优势

1. **突破脉宽限制**:
   - 传统设计受限于边沿检测模块
   - v5设计直接采样，无最小脉宽限制

2. **全范围高精度**:
   - 0-50ns范围内保持35ps精度
   - 无需切换测量模式

3. **高可靠性**:
   - 多级错误检测
   - 亚稳态防护
   - 边界情况处理

4. **易于扩展**:
   - 模块化设计
   - 可添加更多延迟链
   - 支持多通道测量
